As 3 Visões do Dashboard Tableau (Voz Ativa)

Estrutura de Storytelling

O dashboard não é apenas um conjunto de gráficos. É uma jornada guiada que leva o gestor de "Como foi o dia?" (Visão Executiva) para "Por que foi assim?" (Visão do Cliente) e finalmente para "O que fazer?" (Visão de Qualidade).

1️⃣ Visão Executiva

O Pulso da Operação (Topo)

  • BANs: Volume, Taxa de Resolução, TMA, Score RN623
  • Gráfico de Linha: Volume + Sentimento Geral ao longo do tempo
  • Objetivo: Responder "Como foi o dia?" em 5 segundos

2️⃣ Visão do Cliente

A Jornada e o Sentimento (Meio)

  • Sankey/Slope: Variação de Sentimento (Frustrated → Satisfied)
  • Barras Horizontais: Top Temas coloridos por % sentimento negativo
  • Dispersão: Duração vs Silêncio, bolhas por Status Resolução

3️⃣ Visão de Qualidade

Conformidade e Ação (Base)

  • Heatmap: Agentes vs Conformidade RN623 (Azul/Verde = Bom, Laranja/Vermelho = Alerta)
  • Tabela Detalhes: Drill-down com Resumo, Diálogo, Justificativas
  • Objetivo: Atuação cirúrgica nos gargalos

Campos Calculados Essenciais (A Inteligência do Negócio)

Estes campos transformam dados brutos em insights

Cada campo calculado é uma regra de negócio traduzida em lógica. Eles permitem que o Tableau entenda o contexto da saúde suplementar e da conformidade RN623.

Taxa de Resolução (%)

Mostra a eficácia da operação em resolver o problema no primeiro contato (FCR). É crítico para saúde suplementar: quanto mais resolvido na primeira ligação, menor o custo operacional.

SUM(IF [Status Resolução] = 'resolved' THEN 1 ELSE 0 END) / COUNT([Protocolo])

Jornada Emocional (Reversão de Sentimento)

Este cálculo é fantástico para o storytelling. Categoriza o esforço do agente em acalmar o cliente. Mostra se uma chamada que começou "Frustrated" terminou "Satisfied" (Reversão Positiva).

IF [Sentimento Inicial] = 'Frustrated' AND [Sentimento Final] = 'Satisfied' THEN 'Reversão Positiva (Sucesso)' ELSEIF [Sentimento Inicial] = 'Frustrated' AND [Sentimento Final] = 'Frustrated' THEN 'Atrito Mantido (Alerta)' ELSEIF [Sentimento Inicial] = 'Curious' AND [Sentimento Final] = 'Satisfied' THEN 'Atendimento Padrão' ELSE 'Outros' END

Classificação de Risco (RN623)

Facilita a coloração dos gráficos de conformidade. Identifica rapidamente quais chamadas representam risco regulatório para a empresa.

IF [Nível Conformidade] = 'Conforme' THEN 'Baixo Risco' ELSEIF [Nível Conformidade] = 'Não Conforme' AND [Score RN623 (%)] < 50 THEN 'Risco Crítico' ELSE 'Risco Moderado' END

Taxa de Silêncio (%)

Mede o atrito na ligação. Um percentual alto pode indicar lentidão no sistema ou agente com dúvidas. Acima de 30% é considerado crítico.

SUM([Silêncio (s)]) / SUM([Duração (s)])

Interatividade e Navegação (Dashboard Actions)

O Storytelling é Guiado, Não Estático

As Dashboard Actions permitem que o gestor explore os dados de forma intuitiva. Clica em um tema, todos os gráficos se atualizam. Passa o mouse, vê o detalhe da conversa.

1. Filtro de Contexto (Drill-down)

Como Funciona:
O gráfico de "Top Temas" atua como filtro (Use as Filter). Quando o gestor clica na barra de "DIFICULDADE_ACESSO_REDE", todos os KPIs, o Heatmap de agentes e a Tabela de Detalhes se atualizam instantaneamente.

2. Destaque (Highlight) na Jornada

Como Funciona:
Uma Highlight Action liga o "Sentimento Final" ao "Gráfico de Dispersão (Duração vs Silêncio)". Ao passar o mouse sobre "Frustrated", o gráfico acende apenas as bolhas dessas chamadas, revelando se as chamadas frustrantes são também as mais longas.

3. Viz in Tooltip (O Segredo para Telas Limpas)

Como Funciona:
Uma planilha separada com campos de texto longo (Resumo, Diálogo, Justificativas) é inserida na Tooltip do gráfico de dispersão. O gestor não precisa ler blocos de texto o tempo todo. A tela fica limpa com gráficos. Quando vê um ponto fora da curva, passa o mouse e uma "janela" se abre mostrando o resumo e o diálogo exato gerado pela IA.

Dicas Finais de Design para o Gestor (UX/UI)

Regra dos 5 Segundos

O gestor deve entender se a operação está bem ou mal em 5 segundos.

  • Use Verde/Azul para métricas saudáveis
  • Use Vermelho/Laranja para ofensores
  • Evite gráficos de pizza com muitas fatias
  • Prefira gráficos de barras horizontais

Títulos Dinâmicos

Em vez de títulos estáticos, use títulos que contam histórias.

  • ❌ "Análise de Temas"
  • ✓ "Quais temas geram mais frustração nos beneficiários?"
  • ❌ "Conformidade"
  • ✓ "Quais agentes precisam de treinamento em RN623?"

Ocultar o Ruído

Deixe dados granulares ocultos ou na parte inferior.

  • Executivos querem tendências primeiro
  • Detalhes apenas quando necessário
  • Use abas ou expandir/colapsar
  • Tabela de dados granulares (CPF, Protocolo) fica escondida

Conclusão: Do Operacional para o Estratégico

A Evolução do Dashboard

Dashboard Tradicional: Responde "Como foi o dia?" com números exatos. É a base sólida, auditável e confiável. Essencial para compliance com ANS.

Dashboard com IA (Voz Ativa): Responde "Por que foi assim?" com insights profundos. Transforma dados brutos em narrativas que guiam decisões cirúrgicas. É o diferencial competitivo, mas exige calibração e validação contínua.

Juntos: Criam uma experiência de Storytelling onde o gestor não apenas "olha números", mas entende o comportamento do beneficiário e pode atuar com precisão nos gargalos da operação.
